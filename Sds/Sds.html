Software Design Specification

Introduction:

This document will provide a basic guideline to help develop a document storage system as per the requirements stated by PERL International. The document contains different sections to describe the overall working procedure to be maintained throughout the course of development. Section 2 will start with describing the document conventions while section 3.1 and 3.2 will discuss about the design components and a high-level component design. Section 4.1 amd 4.2 contains class diagrams and sequence diagram that are going to be used to develop the project and section 5 will contain the database design followed by section 6 which will contain the implementation plan.

Document Conventions:

For the Class Diagrams, Sequence Diagrams and ER diagrams, seperate legends will be provided in the respective sections. Other than that, this document uses the following conventions: Emphasized word - word, Quoted Text- "qoute".
